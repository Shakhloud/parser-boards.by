# Парсер рекламных поверхностей boards.by

Парсер собирает данные о рекламных конструкциях с сайта [boards.by](https://boards.by), нормализует их и экспортирует в JSON и Excel.

## Стек

- **Python 3.10+** — язык реализации
- **SQLAlchemy 2.0** — ORM для работы с БД
- **PostgreSQL** — база данных для хранения промежуточных результатов
- **Alembic** — миграции базы данных
- **Pydantic** — валидация и сериализация данных
- **BeautifulSoup4** — парсинг HTML
- **Requests** — HTTP-запросы
- **openpyxl** — экспорт в Excel
- **python-dotenv** — загрузка переменных окружения

## Архитектура

```
parse.py
├── Загрузка главной страницы boards.by
├── Извлечение JSON из data-banners (+-2398 объектов)
├── Для каждой конструкции:
│   ├── Парсинг карточки (размер, тип отображения)
│   └── Разбиение на стороны (А, Б, А1...)
├── Нормализация данных:
│   ├── display_type (Призматрон, Скроллер, Видеоэкран, Статика)
│   ├── construction_format (Билборды, Арки, Мосты, Брандмауэры, Ситиборды, Сити-форматы, Нетиповые форматы)
│   └── размер (2,0х6,0 м. → 2.0x6.0)
├── Сохранение в PostgreSQL
└── Экспорт в result.json и result.xlsx
```

## Подготовка окружения

### 1. Разверните PostgreSQL

**База данных уже должна быть запущена.** Рекомендуется использовать Docker:

```bash
docker run -d \
  --name postgres-boards \
  -e POSTGRES_USER=marat \
  -e POSTGRES_PASSWORD=12345678 \
  -p 5432:5432 \
  postgres:16
```

Или используйте существующий инстанс PostgreSQL.

### 2. Установите зависимости

```bash
pip install -r requirements.txt
```

### 3. Настройте переменные окружения

```bash
# Скопируйте пример
cp .env.example .env

# Отредактируйте .env — укажите ссылку для подключения к вашей БД
nano .env
```

Пример `.env`:

```env
# Ссылка для подключения к PostgreSQL
DATABASE_URL=postgresql://user:password@localhost:5432/dbname

# Количество конструкций для парсинга (0 = все)
MAX_CONSTRUCTIONS=100

# Задержка между запросами (секунды)
REQUEST_DELAY=0.02

# Имена выходных файлов
OUTPUT_JSON=result.json
OUTPUT_XLSX=result.xlsx
```

### 4. Примените миграции

```bash
alembic upgrade head
```

Это создаст таблицы `constructions` и `construction_sides` в базе данных.

## Запуск парсера

```bash
python parse.py
```

Парсер:
1. Загрузит данные с boards.by
2. Распарсит карточки для получения размера и типа отображения
3. Сохранит в БД
4. Экспортирует в `result.json` и `result.xlsx`

## Нормализация

### display_type

| Исходное значение | Нормализованное |
|-------------------|-----------------|
| Призматрон, Призмавижн | Призматрон |
| Скролл | Скроллер |
| LED - экран, Светодиодный экран, Экран | Видеоэкран |
| (не определено) | Статика |

### construction_format

| Исходное значение | Нормализованное |
|-------------------|-----------------|
| Билборд | Билборды |
| Арка | Арки |
| Путепровод | Мосты |
| Брандмауэр | Брандмауэры |
| Ситиборд | Ситиборды |
| Световой короб | Сити-форматы |
| Юнипол, Мегаборд, (остальные) | Нетиповые форматы |

### Размер

- `2,0х6,0 м.` → `2.0x6.0`
- `8 х 4 м.` → `8x4`
- `6*3м` → `6x3`

## Ограничения

- **lighting** и **material** не доступны на сайте boards.by — всегда `null`
- **size** и **display_type** парсятся из HTML карточек, возможны пропуски при изменении структуры сайта
- Для парсинга всех 2398 конструкций потребуется ~10-15 минут (с задержкой между запросами)
